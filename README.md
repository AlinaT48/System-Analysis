# System-Analysis

This repository is intended to store and maintain complete documentation for the information system. It serves as a single source of truth for all key project artifacts required for system development, maintenance, and understanding.

## Repository Structure

The repository contains the following documentation sections:

### Non-Functional Requirements (NFR)

Documentation of system quality attributes and constraints, including performance, security, reliability, scalability, and availability requirements.

### Architecture 

The Architecture folder contains system architecture documentation based on the C4 model:
Context.png — System Context diagram showing external users and systems.
Containers.png — Container diagram describing high-level system components and their interactions.
Components.png — Component diagram detailing internal structure of key containers.
Course IBM - C4.txt — code of the C4 model.
Course IBM - Architecture .txt — justification and reasoning behind the selected architectural approach.
   
### Database 

The DB folder contains database-related documentation and artifacts:
Course IBM - ERD.pdf — Entity-Relationship Diagram describing the database structure.
Script 1.txt — database script
Script 2.txt — database script 

### API Documentation 

The API folder contains documentation related to system interfaces and interaction with external clients and services:
Endpoints.xlsx — a structured list of API endpoints, including methods, parameters and descriptions.
Postman — a collection of Postman requests used for API testing and validation.
Swagger.txt — textual description related to the Swagger / OpenAPI specification of the system.

### README

The README file provides an overview of the repository structure, navigation guidelines, and instructions on how to work with the documentation.

### Purpose of the Repository

The purpose of this repository is to provide a single source of truth for all system documentation, ensuring transparency, consistency, and ease of understanding for developers, analysts, and stakeholders.
